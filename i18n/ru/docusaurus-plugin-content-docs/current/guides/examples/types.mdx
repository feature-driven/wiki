# Типы

<WIP ticket="179" />

> **Типы бывают разные**
>
> Это может быть как служебный type (`Maybe`), так и по предметной области (Tls, Contract, Camera, ...)
> 
> 1) То что относится к предметной области обычно лучше класть в shared/api (особенно когда есть кодогенерация в одно место)
>   (по entities такое распылять не очень выйдет)
>
> 2) Общие служебные  типы, которые можно задекларировать - в тот же react-app-env.d.ts
>
> 3) А если у тебя служебные типы и надо импортировать прям везде - то некритично особо, можно и в shared, можно и рядом с react-app-env.d.ts положить
>
> https://t.me/feature_sliced/3879
>
> 4) Нормализованные типы для сущностей имеет смысл хранить в entities
>
> https://t.me/feature_sliced/4513
